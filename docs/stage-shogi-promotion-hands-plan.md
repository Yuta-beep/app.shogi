# Stage Shogi: 昇格・持ち駒 実装計画（特殊スキル後回し）

作成日: 2026-03-10  
スコープ: `stage-shogi` の通常将棋ルール（昇格・持ち駒）  
非スコープ: 特殊スキル、演出、独自ルール拡張

---

## 1. 目的

現状の「盤上移動 + AI応手」MVPを、通常将棋の最低限成立ラインまで引き上げる。

1. 昇格を正しく扱える
2. 持ち駒を保持・表示・打てる
3. その状態を AI リクエスト（SFEN/合法手）に反映できる

---

## 2. 完成条件（ユーザー視点）

- 成れる手で移動すると、`成る / 成らない` を選択できる（必成は強制成り）
- 駒を取ると自分の持ち駒に入る
- 持ち駒エリアから盤面に打てる
- AI側の打ち手・成り手も盤面と持ち駒に正しく反映される
- 王/玉を取るまでループ継続

---

## 3. 状態設計

## 3.1 追加状態

- `hands.player`: `pieceCode -> count`
- `hands.enemy`: `pieceCode -> count`
- `boardPiece.promoted`: `boolean`
- `pendingPromotion`: 成り選択待ち状態

## 3.2 UI状態

- `player_turn.idle`
- `player_turn.piece_selected`
- `player_turn.drop_selected`（持ち駒を選択中）
- `player_turn.promotion_confirm`
- `ai_thinking`
- `finished`

---

## 4. UI要件

## 4.1 持ち駒エリア

- 盤外に先手/後手の持ち駒表示領域を追加
- 先手ターン中のみ自分の持ち駒をタップ可能
- 打てない駒は disabled 表示

## 4.2 盤面操作

- 盤上の自駒選択時: 移動合法手ハイライト
- 持ち駒選択時: 打ち合法手ハイライト
- 非合法マスは無効（タップしても確定しない）

## 4.3 昇格UI

- 任意成り: モーダル `成る / 成らない`
- 必成（歩/香の最終段、桂の最終2段）はモーダルなしで自動成り

---

## 5. ルール要件（特殊スキルなし）

## 5.1 昇格

- 対象: `FU, KY, KE, GI, KA, HI`
- 既に成っている駒は再成り不可
- 成り駒の移動は通常将棋準拠

## 5.2 取る/持ち駒

- 取った駒は成りを解除し、取り手側の持ち駒へ加算
- 王/玉は持ち駒に入れない（終局判定優先）

## 5.3 打ち

- 盤外から空きマスへ配置
- 禁止:
  - 二歩
  - 行き所のない駒打ち（歩/香の最終段、桂の最終2段）
  - 打ち歩詰め（可能なら実装、難しければ Phase2 で厳密化）

---

## 6. データ/API要件

## 6.1 盤面モデル

- `BoardPiece` に `promoted` を追加
- `hands` はローカル状態として保持し、AI適用時も更新

## 6.2 SFEN生成

- 盤面: 成り駒を `+` 付きで出力
- 手番: `b / w`
- 持ち駒: `-` 固定を廃止し、実際の持ち駒を出力

## 6.3 legalMoves生成

- `legalMoves` を空配列固定で送らない
- 移動手・打ち手・成り手を含める
- `fromRow/fromCol` が `null` の場合は打ち手

## 6.4 AI応答適用

- `promote=true` を盤面状態へ反映
- `capturedPieceCode` を使って持ち駒更新
- `dropPieceCode` がある場合は持ち駒を減算して盤面配置

---

## 7. 実装ステップ

1. ドメイン関数拡張  
`game-rules` に昇格判定・持ち駒更新・打ち合法判定を追加

2. 画面状態拡張  
`stage-shogi-screen` に `hands` / `pendingPromotion` / `drop` 操作を導入

3. 表示拡張  
持ち駒UI、成り確認UIを追加

4. 送受信拡張  
SFEN/legalMoves/AI応答適用を昇格・持ち駒対応へ変更

5. テスト追加  
下記テスト要件を満たす

---

## 8. テスト要件（必須）

## 8.1 Unit（`game-rules`）

- 成り可能判定（任意成り/必成）
- 取った駒が未成で持ち駒化される
- 持ち駒打ち合法判定（二歩・行き所なし）
- 打ち手/移動手で盤面・持ち駒が正しく更新される

## 8.2 Component/Hook（`stage-shogi`）

- 成り確認モーダルで分岐（成る/成らない）
- 持ち駒タップ -> 盤面タップで打てる
- 非合法マスでは確定しない
- 着手後に `/api/v1/ai/move` が呼ばれる

## 8.3 API payload test（フロント側）

- 昇格状態を含む SFEN を生成できる
- `legalMoves` に打ち手と成り手が含まれる
- `hands` と盤面の整合が送信前に保たれる

## 8.4 回帰テスト

- 既存の「プレイヤー着手 -> AI応手 -> 継続」が壊れていない

---

## 9. 受け入れ条件（DoD）

1. 昇格・持ち駒の基本操作がUI上で完結する
2. AI応答で成り/打ちが返ってきても局面が破綻しない
3. `bun run test` で追加テストを含めて green
4. `bun run lint` / `bun run typecheck` が green
5. 特殊スキルは未実装でも、通常将棋ルールとして対局継続可能

---

## 10. 明示的な後回し項目

- 特殊スキル発動
- スキル専用アニメーション/演出
- 将棋以外の独自効果（バフ、状態異常など）
- AIの特殊スキル評価ロジック
